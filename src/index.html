W-Pay JS SDK Examples

Basic Example

The basic example page follows the standard payments flow. Tokenizing a card, creating a new payment request and making a payment using the new payment instrument. The code for this example has been thoroughly comment to help explain what is happening.

Open Basic Example

Client Example

This example represents the flow that would be followed by a client using a merchant website to make a payment. It follows the immediate payments flow which does not require the merchant to initialise a new payment request and includes a configuration panel on the page allowing you to turn 3DS checking or switch between making a Payment or a Preauth transaction. The page also includes a code examples window that shows the SDK methods called as part of each step.

The transactionId can be copied from the output of this page over to the Merchant Service Portal Example in order to process refunds, completions and voids.

Open Client Example

Merchant Service Portal Example

This example represents an internal page for the Merchant where they can process customer refunds, completions and voids. This functionality should only be initiated from the merchant side and never directly from a customer facing page. The Merchant Service Portal Example includes the same code example window as the Client Example to show the SDK methods being used.
